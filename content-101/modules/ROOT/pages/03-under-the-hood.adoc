== Exercise 2: Three Models, Same Batch

Run all 10 tickets through each model. The script scores each model against a ground truth key:

[source,bash,role="execute",subs=attributes+]
----
TICKETS=(
  "My invoice shows a charge for a service I cancelled last month."
  "The dashboard won't load — I get a 502 error every time."
  "How do I add a second user to my organization?"
  "Your platform saved us 200 hours last quarter. Thank you!"
  "I was charged twice for the same subscription period."
  "API responses are timing out after the latest update."
  "Can I transfer ownership of my account to a colleague?"
  "The new reporting feature is exactly what we needed."
  "My payment method was declined but the card works everywhere else."
  "Pods are stuck in CrashLoopBackOff after upgrading to 4.16."
)
TRUTH=(billing technical account praise billing technical account praise billing technical)

for model in granite-2b-cpu qwen25-3b-cpu phi3-mini-cpu; do
  echo "=== $model ==="
  correct=0
  total_ms=0
  start_all=$(date +%s%3N)
  for i in $(seq 0 9); do
    RESULT=$(curl -s -X POST {maas_url}/v1/chat/completions \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer {litellm_api_key}" \
      -d "{\"model\": \"$model\", \"messages\": [{\"role\": \"user\", \"content\": \"Classify this support ticket into exactly one category: billing, technical, account, or praise. Respond with only the category name.\n\nTicket: ${TICKETS[$i]}\"}], \"max_tokens\": 10}")
    answer=$(echo "$RESULT" | jq -r '.choices[0].message.content' | tr '[:upper:]' '[:lower:]' | tr -d ' .')
    if [ "$answer" = "${TRUTH[$i]}" ]; then
      correct=$((correct + 1))
    fi
    echo "  Ticket $((i+1)): predicted=$answer truth=${TRUTH[$i]}"
  done
  end_all=$(date +%s%3N)
  elapsed=$((end_all - start_all))
  echo "  Accuracy: $correct/10  Total time: $elapsed ms"
  echo ""
done
----

This takes 1-2 minutes. Watch the results stream in.

* Different models have different accuracy/speed tradeoffs on the same task
* A 2B model can often match a 3.8B model on simple classification -- model selection matters
* CPU inference at $0/token means you can run experiments and batch jobs without worrying about API bills
* You now have a data-backed opinion about which model to deploy -- that is Module 4
