= Module 4: Make It Yours
:navtitle: Make It Yours

== Objective

Test the agent with real scenarios, then write your own system prompt for a customer you know. Deploy it, test it, and leave with an agent configured for your territory.

== Exercise 1: Three Customer Scenarios

Run a financial services scenario:

[source,bash,role="execute",subs=attributes+]

== Exercise 2: Write Your Prompt

The system prompt is the agent's expertise. You are going to replace it with one tailored to a customer you know.

First, read the current prompt:

[source,bash,role="execute",subs=attributes+]
----
oc get configmap advisor-prompt -o jsonpath='{.data.system_prompt}'
----

Now write your own. Pick a customer or industry you work with and fill in this template:

[source,bash,subs=attributes+]
----
# Edit this prompt for YOUR customer scenario, then run the block below to deploy it.
#
# Replace the bracketed sections with specifics from your territory:
# - Who is the customer? (industry, size, region)
# - What is their pain point? (cost, speed, compliance, modernization)
# - What should the agent emphasize? (edge, security, migration, TCO)
----

Deploy your custom prompt:

[source,bash,role="execute",subs=attributes+]
----
cat << 'PROMPT_EOF' | oc create configmap advisor-prompt --from-literal=system_prompt="$(cat)" --dry-run=client -o yaml | oc apply -f -
You are a Red Hat + Intel Solution Architect specializing in helping enterprise customers modernize their AI infrastructure.

When a customer describes their workload, follow this process:
1. Identify the workload type and key constraints
2. Look up matching Intel hardware using the intel_hardware_lookup tool
3. Check which OpenShift operators and capabilities apply using openshift_capabilities
4. Find a reference architecture using reference_architectures
5. Generate a solution brief

Your recommendation must include:
- Executive Summary (2-3 sentences)
- Recommended Intel Hardware by deployment tier
- OpenShift Platform Components with specific operator names
- Architecture Overview showing how the pieces connect
- Cost Considerations comparing on-premises vs cloud
- Why Red Hat + Intel Together — the specific integration advantages

Always cite specific product names (Xeon 6, Gaudi, OpenShift AI, AMQ Streams).
Always include a "Migration Path" section for customers on legacy infrastructure.
Keep the brief under 800 words.
PROMPT_EOF
----

Restart the agent to pick up the new prompt:

[source,bash,role="execute",subs=attributes+]
----
oc rollout restart deploy/solution-agent
oc rollout status deploy/solution-agent --timeout=90s
----

== Exercise 4: Swap the Model

Try a larger model to see if the recommendation quality improves:

[source,bash,role="execute",subs=attributes+]
----
oc set env deploy/solution-agent ADVISOR_MODEL=phi3-mini-cpu
oc rollout status deploy/solution-agent --timeout=90s
----

Re-run your customer scenario:

[source,bash,subs=attributes+]
----
curl -s -X POST {advisor_url}/api/v1/advise \
  -H "Content-Type: application/json" \

== Learning Outcomes

* The system prompt is the primary control surface — change the persona, process, and output format without touching code
* Prompt engineering is iterative: write, test, observe, revise
* Model selection is a quality/latency tradeoff — test both, pick based on your use case
* Every change is a ConfigMap or env var — standard Kubernetes operations that any platform team can manage
* You leave with an agent configured for YOUR customer scenario
