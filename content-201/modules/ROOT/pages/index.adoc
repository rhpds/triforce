= Building an AI Agent — Hands-On Lab
:navtitle: Welcome

== What You'll Build

In this lab you will build, deploy, and tune an AI agent on OpenShift. The *Red Hat + Intel Solution Architect Agent* takes customer workload requirements, looks up Intel hardware specs and OpenShift capabilities via MCP tools, and generates a joint solution brief.

By the end, you will understand what makes an agent different from a model, and how to deploy one as standard Kubernetes infrastructure.

== Learning Objectives

* Understand the difference between a model (answers questions) and an agent (solves problems)
* Deploy MCP tools that provide structured data to an agent
* Deploy a LangGraph agent with a configurable system prompt
* Test the agent with real customer scenarios
* Tune agent behavior by swapping models and editing prompts

== Why This Matters for You

As a tech seller, you field workload sizing questions daily. This agent does the product lookup, benchmark analysis, and platform recommendation work. You validate the output and present it to the customer.

More broadly: this is how enterprise AI gets built. Not one monolithic model, but a model wrapped in tools and orchestration. This pattern applies to any domain.

== The Agent Pattern

```
Customer Workload Description
          │
          ▼
   ┌──────────────┐
   │ System Prompt │  ← "You are a Red Hat + Intel Solution Architect..."
   │  (persona)    │
   └──────┬───────┘
          │
   ┌──────┴───────┐
   │   LangGraph   │  ← Multi-step workflow
   │ (orchestration)│
   └──────┬───────┘
          │
     ┌────┼────┐
     │    │    │
   Intel  OCP  Ref
   Specs  Caps Archs
   (MCP)  (MCP) (MCP)
     │    │    │
     └────┼────┘
          │
          ▼
   Solution Brief
   (with citations)
```

In 101 you deployed a model. A model answers questions. An *agent* solves problems.
