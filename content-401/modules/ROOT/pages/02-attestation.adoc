= Attestation Handshake
:navtitle: Attestation

== How It Works

Before the AI agent gets its API key, the hardware must prove it's genuine:

```
Pod starts → Kata VM boots inside TDX Trust Domain
     → CDH agent contacts Trustee KBS
     → vTPM generates attestation quote
     → Trustee verifies hardware state
     → Secrets released into Trust Domain
```

No TDX = no attestation = no secrets = agent can't start.

== Exercise 1: Check Trustee KBS

[source,bash,role="execute"]
----
oc get pods -n trustee-operator-system
oc get kbsconfig -n trustee-operator-system -o yaml | head -20
----

The KBS holds your API keys. It only releases them after verifying TDX attestation.

== Exercise 2: Check Agent Attestation Status

[source,bash,role="execute"]
----
oc logs deploy/healthcare-agent -n triforce | grep -i "tdx\|attestation\|trust domain\|CDH"
----

If running inside TDX: `"Running inside TDX Trust Domain — inference data is hardware-encrypted"`
If standard mode: `"Running in standard mode (no TDX)"`

== Exercise 3: Review Secret Policy

[source,bash,role="execute"]
----
oc get secret triforce-secrets -n triforce -o json | jq '.data | keys'
----

These secrets are only accessible to attested workloads when `confidential.enabled=true`.
