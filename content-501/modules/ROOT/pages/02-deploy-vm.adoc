= See the VM Running
:navtitle: Deploy VM

== Exercise 1: Check the Virtual Machine

[source,bash,role="execute"]
----
oc get vm legacy-database -n triforce -o jsonpath='{.status.printableStatus}'
----

The VM is a RHEL 9 guest image running as a KubeVirt VirtualMachine — a Kubernetes CRD.

== Exercise 2: VM Instance Details

[source,bash,role="execute"]
----
oc get vmi legacy-database -n triforce -o json | jq '{name: .metadata.name, node: .status.nodeName, phase: .status.phase, cpu: .spec.domain.cpu, memory: .spec.domain.resources}'
----

4 cores, 8Gi memory — a typical legacy database workload. Running on the same Xeon 6 node as the AI agents.

== Exercise 3: What's on the Same Node

[source,bash,role="execute"]
----
NODE=$(oc get vmi legacy-database -n triforce -o jsonpath='{.status.nodeName}')
echo "Node: $NODE"
oc get pods -n triforce --field-selector spec.nodeName=$NODE -o wide
----

VMs and AI agent containers — same node, same CPU, same $0 cost.
