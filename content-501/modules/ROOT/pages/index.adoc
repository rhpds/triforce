= VMs + AI — Hands-On Lab
:navtitle: Welcome

== Welcome

You have hundreds of VMs running legacy databases, middleware, and enterprise applications. You can't containerize them overnight. But AI can't wait for your migration to finish.

*What if both could run on the same hardware?*

OpenShift Virtualization runs your VMs as first-class Kubernetes resources. The same Intel Xeon 6 servers running your VMs today can run AI inference tomorrow — on the same node, managed by the same platform.

== What You'll Do

=== Required (~20 minutes)

1. **Verify the platform** — OpenShift Virtualization operator, HyperConverged resource
2. **See the VM running** — legacy RHEL VM alongside AI agent containers
3. **Prove coexistence** — both workload types on the same Xeon 6 node
