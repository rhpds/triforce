= Scale Test
:navtitle: Scale Test

== Learning Objectives

* Push synthetic data through the Kafka pipeline at scale
* Measure throughput and latency under concurrent load
* See how the platform handles multiple records simultaneously

== Exercise 1: Push Synthetic Data Through Kafka

. Generate and push 5 patient records through Kafka:
+
[source,bash,role="execute",subs=attributes+]
----
python3 -c "
import httpx, json
from synthetic.healthcare_generator import generate_batch

records = generate_batch(5)
for r in records:
    resp = httpx.post('http://redpanda:18082/topics/healthcare.patients.synthetic',
        headers={'Content-Type': 'application/vnd.kafka.json.v2+json'},
        json={'records': [{'value': r}]}, timeout=5.0)
    print(f'  [{r[\"record_type\"]}] offset={resp.json()[\"offsets\"][0][\"offset\"]}')
print('Done — 5 records pushed')
"
----

. Wait 30 seconds, then check processing:
+
[source,bash,role="execute"]
----
sleep 30
oc logs deploy/healthcare-agent -n triforce --tail=10 | grep "Processed patient"
----

== Exercise 2: Run the Scale Test Runner

. Navigate to the *Scale Test* page in the frontend (if available), or run from the terminal:
+
[source,bash,role="execute"]
----
python3 /opt/triforce/synthetic/poc_scale_harness.py --concurrency 1,3 --records 5
----

. Review the output:
** Throughput at different concurrency levels
** Latency percentiles (avg, p50, p95)
** Healthcare: ~0.1-0.2 records/sec (multi-step LangGraph pipeline)
** FinServ: ~100 records/sec (rule-based scoring)
