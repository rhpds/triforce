= Module 5: When CPU Isn't Enough
:navtitle: When CPU Isn't Enough

== Objective

You have right-sized your models and applied optimizations. Now find where CPU hits its limit and GPU acceleration is worth the cost.

== Exercise 1: Classification -- CPU vs GPU

Compare the fastest CPU model against a GPU model on classification:

[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {healthcare_url}/api/v1/benchmark/run \
  -H "Content-Type: application/json" \
  -d '{"task": "classification", "text": "DISCHARGE SUMMARY: 72-year-old male with Type 2 Diabetes on Metformin 500mg.", "models": ["granite-2b-cpu", "granite-3-2-8b-instruct"]}' \
  | jq '{task, results: [.results[] | {model, hardware, latency_ms, output: .output[:30]}]}'
----

== Exercise 3: Complex Reasoning -- Where CPU Falls Over

Try a task that requires multi-step reasoning on the 8B model -- CPU vs GPU:

[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {healthcare_url}/api/v1/benchmark/run \
  -H "Content-Type: application/json" \
  -d '{"task": "compliance_reasoning", "text": "A customer in Germany transfers $9,500 to a business account in the Cayman Islands. They have made 3 similar transfers in the past month, each just under $10,000. The customer is a small business owner with no known international operations. Their average monthly outbound transfer volume is $15,000. Analyze this pattern for regulatory compliance.", "models": ["granite-3-2-8b-instruct-cpu", "granite-3-2-8b-instruct"]}' \
  | jq '{task, results: [.results[] | {model, hardware, latency_ms, output_tokens}]}'
----

[sidebar]
****
*CPU (8B):* ___ ms | *GPU (8B):* ___ ms | *Speedup:* ___x

*Same model, different hardware. Where is the break-even?*
****

NOTE: If GPU models return errors, it means Gaudi accelerators are not available on this cluster. Use the CPU-only results to extrapolate -- GPU models on Intel Gaudi 3 typically run 2-4x faster for generation-heavy tasks based on measured benchmarks.
