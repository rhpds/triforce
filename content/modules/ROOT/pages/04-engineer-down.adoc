== Objective

Apply four optimization techniques that make CPU inference faster -- without changing models. Measure each one and calculate how they compound.

== Exercise 1: Adaptive Caching

The adaptive cache learns from LLM results. After seeing a document's text hash once, it returns the cached classification in <1ms instead of calling the LLM.

First, reset the cache so you start clean:

[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {healthcare_url}/api/v1/adaptive/reset | jq .
----

Check the cache stats -- should be empty:

[source,bash,role="execute",subs=attributes+]
----
curl -s {healthcare_url}/api/v1/adaptive/stats | jq .
----

Now run the same clinical text 5 times and watch the cache learn:

[source,bash,role="execute",subs=attributes+]
----
for i in 1 2 3 4 5; do
  echo "--- Run $i ---"
  curl -s -X POST {healthcare_url}/api/v1/pipeline \
    -H "Content-Type: application/json" \
    -d '{"text": "DISCHARGE SUMMARY: 72-year-old male with Type 2 Diabetes on Metformin 500mg."}' \
    | jq '{run: '$i', classify_ms: (.inference_log[] | select(.node=="classify") | .latency_ms), cache_hit: (.inference_log[] | select(.node=="classify") | .kv_cache_hit)}'
done
----

Check the cache stats again:

[source,bash,role="execute",subs=attributes+]
----
curl -s {healthcare_url}/api/v1/adaptive/stats | jq '{cache_size, hit_rate, llm_reduction_pct}'
----

****
*Record:*

* Run 1 classify latency (cold): ___ ms
* Run 2 classify latency (cached): ___ ms
* Speedup from caching: ___x
* Hit rate after 5 runs: ___%
****

== Exercise 3: Semantic Routing

The semantic router classifies request complexity in <1ms using embeddings -- no LLM call. It decides which model should handle the request.

Send a simple query:

[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {semantic_router_url}/classify \
  -H "Content-Type: application/json" \
  -d '{"text": "What type of document is this?"}' \
  | jq '{route, model, confidence, latency_ms, method}'
----

Send a complex query:

[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {semantic_router_url}/classify \
  -H "Content-Type: application/json" \
  -d '{"text": "Analyze the drug interactions between Warfarin and Aspirin considering the patients renal function and provide a differential diagnosis"}' \
  | jq '{route, model, confidence, latency_ms, method}'
----

* Complex query → route: ___, model: ___, latency: ___ ms

=== Misrouting: When the Router Gets It Wrong

Now send 5 texts of varying complexity and record the route for each. Some of these are trivially simple, some are moderately complex, and one is deliberately dense:

[source,bash,role="execute",subs=attributes+]
----
for text in \
  "What type of document is this?" \
  "Patient on Metformin." \
  "Evaluate the interaction profile of concurrent administration of Warfarin and Aspirin in a patient with compromised hepatic function and recent mechanical valve replacement" \
  "A patient has a headache." \
  "Summarize this note briefly."; do
  echo "=== Text: ${text:0:50}..."
  curl -s -X POST {semantic_router_url}/classify \
    -H "Content-Type: application/json" \
    -d "{\"text\": \"$text\"}" \
    | jq '{route: .route, model: .model, confidence: .confidence}'
done
----

== Exercise 4: MCP Tools vs LLM

Some questions don't need an LLM at all. The MCP Gateway provides 8 tools that answer with deterministic database lookups.

Call the drug interaction tool directly via JSON-RPC:

[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {mcp_gateway_url}/mcp \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc": "2.0", "id": 1, "method": "tools/call", "params": {"name": "drug_interaction_check", "arguments": {"medications": ["Warfarin", "Aspirin", "Metformin"]}}}' \
  | jq .
----

Now compare: how long does the LLM take to answer the same question?

[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {healthcare_url}/api/v1/pipeline \
  -H "Content-Type: application/json" \
  -d '{"text": "Patient on Warfarin, Aspirin, and Metformin. Check for drug interactions.", "skip_cache": true}' \
  | jq '{total_ms, interaction_step: (.inference_log[] | select(.node=="check_interactions") | {tool: .tool, latency_ms})}'
----

== Exercise 5: Compound Savings

Optimizations compound. Caching alone saves milliseconds -- but caching plus conditional skipping plus routing adds up fast. Measure the difference end-to-end.

First, reset the cache so the cold run is truly cold:

[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {healthcare_url}/api/v1/adaptive/reset | jq .
----

Now run 10 pipeline requests cold (no cache), then 10 warm (cache enabled), and compare total wall-clock time:

[source,bash,role="execute",subs=attributes+]
----
echo "=== Cold (no cache) ==="
start=$(date +%s%N)
for i in $(seq 1 10); do
  curl -s -X POST {healthcare_url}/api/v1/pipeline \
    -H "Content-Type: application/json" \
    -d '{"text": "DISCHARGE SUMMARY: 72-year-old male with Type 2 Diabetes on Metformin 500mg.", "skip_cache": true}' -o /dev/null
done
cold_ms=$(( ($(date +%s%N) - start) / 1000000 ))

echo "=== Warm (cache enabled) ==="
start=$(date +%s%N)
for i in $(seq 1 10); do
  curl -s -X POST {healthcare_url}/api/v1/pipeline \
    -H "Content-Type: application/json" \
    -d '{"text": "DISCHARGE SUMMARY: 72-year-old male with Type 2 Diabetes on Metformin 500mg."}' -o /dev/null
done
warm_ms=$(( ($(date +%s%N) - start) / 1000000 ))
echo "Cold: $\{cold_ms}ms  |  Warm: $\{warm_ms}ms  |  Savings: $(( cold_ms - warm_ms ))ms"
----
