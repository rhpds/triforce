= Build: MCP Tool Deep Dive
:navtitle: Build: MCP Tools

*Time: ~18 minutes* · *Terminal*

== What You'll Build

The MCP Gateway federates 8 tools across 3 contract domains -- healthcare (FHIR), financial services (risk/compliance), and platform (observability). Every tool call is a JSON-RPC request that returns structured data in milliseconds, while an equivalent LLM call takes seconds.

You will inventory all 8 tools, call each healthcare tool and measure its latency, construct your own JSON-RPC call to a tool you choose, quantify tool overhead at scale, and design a tool routing strategy for production.

The gateway API is a single endpoint: `POST {mcp_gateway_url}/mcp` with a JSON-RPC body. Two methods matter:

* `tools/list` -- returns all registered tools and their input schemas
* `tools/call` -- invokes a tool by name with arguments

== Exercise 1: Inventory All Tools

List every tool the gateway exposes:

[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {mcp_gateway_url}/mcp \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":"1","method":"tools/list","params":{}}' \
  | jq '.result.tools[] | {name, description}'
----

== Exercise 2: Call Each Healthcare Tool

Call all 3 FHIR tools and record the latency for each. These are data lookups -- no LLM inference involved.

*Drug interaction check* -- 3 medications, checking for pairwise interactions:

[source,bash,role="execute",subs=attributes+]
----
time curl -s -X POST {mcp_gateway_url}/mcp \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":"3","method":"tools/call","params":{"name":"drug_interaction_check","arguments":{"medications":["Warfarin","Aspirin","Metformin"]}}}' \
  | jq '.result.structuredContent'
----

*Patient lookup* -- retrieve a synthetic patient record:

[source,bash,role="execute",subs=attributes+]
----
time curl -s -X POST {mcp_gateway_url}/mcp \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":"4","method":"tools/call","params":{"name":"fhir_patient_lookup","arguments":{"patient_id":"P-001"}}}' \
  | jq '.result.structuredContent | {patient_id, name, age, conditions: [.conditions[].display], medications: [.medications[].name]}'
----

*Clinical code search* -- find ICD-10 codes for a condition:

[source,bash,role="execute",subs=attributes+]
----
time curl -s -X POST {mcp_gateway_url}/mcp \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":"5","method":"tools/call","params":{"name":"clinical_code_search","arguments":{"query":"Type 2 Diabetes","code_system":"icd10"}}}' \
  | jq '.result.structuredContent.results[] | {code, display}'
----

== Exercise 4: Tool vs LLM at Scale

When the pipeline runs with medications, the `check_interactions` step fires an MCP tool call. When there are no medications, that step is skipped. Measure the difference at scale.

First, run 10 requests WITH medications (tools fire):

[source,bash,role="execute",subs=attributes+]
----
echo "=== 10 requests WITH medications (tools fire) ==="
start=$(date +%s%N)
for i in $(seq 1 10); do
  curl -s -X POST {healthcare_url}/api/v1/pipeline \
    -H "Content-Type: application/json" \
    -d '{"text": "Patient on Aspirin 81mg, Clopidogrel 75mg, and Metformin 1000mg. History of Type 2 Diabetes and coronary artery disease.", "skip_cache": true}' \
    -o /dev/null &
done
wait
end=$(date +%s%N)
with_tools_ms=$(( (end - start) / 1000000 ))
echo "  Wall clock: $\{with_tools_ms}ms  |  Avg per request: $(( with_tools_ms / 10 ))ms"
----

Now run 10 requests WITHOUT medications (no tool calls, short text with 0 meds):

[source,bash,role="execute",subs=attributes+]
----
echo "=== 10 requests WITHOUT medications (tools skip) ==="
start=$(date +%s%N)
for i in $(seq 1 10); do
  curl -s -X POST {healthcare_url}/api/v1/pipeline \
    -H "Content-Type: application/json" \
    -d '{"text": "Routine checkup. Vitals normal.", "skip_cache": true}' \
    -o /dev/null &
done
wait
end=$(date +%s%N)
without_tools_ms=$(( (end - start) / 1000000 ))
echo "  Wall clock: $\{without_tools_ms}ms  |  Avg per request: $(( without_tools_ms / 10 ))ms"
----
