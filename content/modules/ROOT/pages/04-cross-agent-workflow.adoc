= Cross-Agent Workflows
:navtitle: Workflows

== Learning Objectives

* Execute a multi-agent workflow that spans healthcare and financial services
* See the Go orchestrator discover agents via A2A and dispatch tasks
* Understand how Kagenti would govern this at scale

== Exercise 1: Discover Agents

. List all agents the orchestrator has discovered:
+
[source,bash,role="execute",subs=attributes+]
----
curl -s {orchestrator_url}/api/v1/agents | jq '.agents[] | {name, status, skills: [.skills[].id]}'
----
+
Expected: Healthcare Agent (classify, extract, summarize) and Financial Services Agent (score, compliance, risk).

== Exercise 2: Run a Cross-Agent Workflow

. Start a `patient_financial_risk` workflow:
+
[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {orchestrator_url}/api/v1/workflows \
  -H "Content-Type: application/json" \
  -d '{
    "workflow_type": "patient_financial_risk",
    "input": {
      "text": "DISCHARGE: 72yo with diabetes, hypertension, CAD. On Metformin, Lisinopril, Atorvastatin.",
      "customer_id": "cust-demo-001"
    }
  }' | jq .
----
+
The orchestrator dispatches to both agents via A2A `tasks/send`.

. Wait 10 seconds for the workflow to complete, then check results:
+
[source,bash,role="execute",subs=attributes+]
----
sleep 10
curl -s {orchestrator_url}/api/v1/workflows | jq '.[] | {id, status, duration_ms, agents_involved, steps: [.steps[] | {agent, status, inference_ms}]}'
----
+
Expected: 2 steps completed (Healthcare Agent + Financial Services Agent), total ~8-10 seconds.

== How It Works

```
Orchestrator (Go)
    ├── A2A tasks/send → Healthcare Agent (Python/LangGraph)
    │   └── classify → extract → interactions → summarize (Xeon 6)
    ├── A2A tasks/send → FinServ Agent (Java/Quarkus)
    │   └── risk assessment (Xeon 6)
    └── Combine results → workflow complete
```

All inference on Intel Xeon 6 CPU. The orchestrator does zero inference — pure coordination.
