= AI Inference on Intel Xeon 6 — Hands-On Lab
:navtitle: Welcome

== What You'll Do

In this lab you will measure, compare, and optimize AI inference workloads running on Intel Xeon 6 CPUs. By the end, you will have a decision framework: given a workload, which model goes on which hardware, and which optimizations make CPU inference fast enough that GPU becomes optional.

You will not watch a demo. You will run the experiments, configure the system, and make the calls yourself.

== Learning Objectives

* Measure inference latency across multiple CPU models on real-world tasks
* Right-size model selection per task and apply your choices to the running system
* Configure optimization techniques (caching, routing, fusion, speculative decoding) and measure their impact
* Decide where CPU is sufficient and where GPU acceleration is worth the cost
* Deploy your optimized stack and compare against baseline

== What's Running

Your environment has 10 pods in a single OpenShift namespace:

[cols="2,1,2",options="header"]
|===
| Service | Language | Role
| Healthcare Agent (×2) | Python / FastAPI | Clinical NLP pipeline -- classify, extract entities, check interactions, summarize
| FinServ Agent (×2) | Java / Quarkus | Fraud scoring -- rule engine + LLM hybrid
| Orchestrator | Go | A2A agent discovery + workflow coordination
| Semantic Router | Python | Embedding-based complexity routing (<1ms after warm-up)
| MCP Gateway | Python | 8 federated tools via JSON-RPC
| Frontend | React / nginx | Visual demo interface
| PostgreSQL | -- | Inference audit trail
| Redpanda | -- | Event streaming (Kafka-compatible)
|===

All inference runs remotely on Intel Xeon 6 via MAAS/LiteLLM. No GPU or model-serving pods are in your namespace -- the models are shared infrastructure.
