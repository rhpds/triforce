= Triforce — AI Inference at Scale on Intel Hardware
:navtitle: Welcome

== What This Lab Is

A hands-on platform for running, measuring, and tuning AI inference across Intel's compute spectrum — Xeon 6 CPU at $0/token for the 80% of tasks that don't need a GPU, and Intel Gaudi GPU for the 20% that do. You have a live multi-agent system deployed on OpenShift — healthcare NLP, financial fraud scoring, semantic routing, MCP tool federation — with 13 models spanning both tiers.

This is not a demo walkthrough. You will call APIs, run experiments, swap models, measure latency, and decide for yourself where CPU is enough and where GPU adds value.

TIP: Looking for the guided demo walkthrough instead? The Triforce demo script is available in the `demo/engineer/` directory of the repo — run `triforce_demo.py` for the full narrated experience with Rich terminal visualization.

== What You Can Explore

[cols="1,3",options="header"]
|===
| Section | What You'll Do

| *Core Platform*
| Call the Healthcare and FinServ APIs directly. Run clinical NLP pipelines, score transactions, discover agents via A2A, use MCP tools — all against live services.

| *Experiments*
| Compare models head-to-head across CPU and GPU. Calculate your break-even vs cloud APIs. Fire concurrent load and watch throughput vs latency. Run the benchmark suite.

| *Optimization*
| Enable the adaptive cache and watch it learn. Try speculative decoding. Run multi-model fusion. Scale replicas and measure improvement.

| *Advanced*
| Explore heterogeneous routing (CPU + GPU per request). Disaggregated inference with llm-d. Edge inference with BitNet on 2-core hardware.
|===

Every exercise produces real numbers from the deployed system. Nothing is hardcoded.
