= llm-d Inference
:navtitle: llm-d Inference

*Time: ~10 minutes* · *Terminal*

== The Question

Can you get sub-2s pipeline latency by splitting prefill and decode across specialized nodes? llm-d separates the two phases of LLM inference — prefill (compute-heavy) and decode (memory-bound) — with SLO-based routing.

== How It Works

Standard inference:
```
One node does everything: prompt processing + token generation + queueing
```

Disaggregated inference (llm-d):
```
Request → llm-d Planner (SLO: 2s latency target)
              │
    ┌─────────┼─────────┐
    │                   │
Prefill Node        Decode Node
(compute-heavy)     (memory-bound)
(128 cores)         (stream tokens)
    200ms               1.2s
              │
         Total: 1.4s (within 2s SLO)
```

== What You Measured

Beyond identical replicas — llm-d specializes nodes for different phases of inference. The prefill burst goes to high-core nodes. Token streaming goes to memory-optimized nodes. The planner routes based on latency SLO targets.

Today's 7.8s pipeline → 1.4s projected with full disaggregation. Same $0/token.
