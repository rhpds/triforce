= Adaptive Cache
:navtitle: Adaptive Cache

*Time: ~10 minutes* · *Terminal*

== The Question

Can the system get cheaper per record over time? The adaptive cache stores every LLM classification result. On re-encounter, it returns the cached answer in <1ms. The improvement compounds with volume.

== Exercise 1: Check Current Cache Stats

[source,bash,role="execute",subs=attributes+]
----
curl -s {healthcare_url}/api/v1/adaptive/stats | jq .
----

This shows how many classifications are cached, the current hit rate, and the LLM reduction percentage.

== Exercise 2: Run the Pipeline and Watch the Cache Learn

Run the same clinical record 5 times. The first call classifies via LLM. Calls 2-5 return from cache:

[source,bash,role="execute",subs=attributes+]
----
for i in $(seq 1 5); do
  echo "=== Call $i ==="
  curl -s -X POST {healthcare_url}/api/v1/pipeline \
    -H "Content-Type: application/json" \
    -d '{"text": "DISCHARGE SUMMARY: 72-year-old male with Type 2 Diabetes on Metformin 500mg and Lisinopril 10mg."}' \
    | jq '{classification, classify_model: .inference_log[0].model, classify_ms: .inference_log[0].latency_ms, cache_hit: .inference_log[0].kv_cache_hit}'
done
----

Call 1: `model: "qwen25-3b-cpu"`, `cache_hit: false`, classify latency ~1000ms. Calls 2-5: `model: "adaptive-cache"`, `cache_hit: true`, classify latency <1ms.

== Exercise 3: Check the Hit Rate Climbing

[source,bash,role="execute",subs=attributes+]
----
curl -s {healthcare_url}/api/v1/adaptive/stats | jq '{cache_size, hit_rate, llm_reduction_pct}'
----

After the 5 runs above, `hit_rate` should be ~0.80 (4 hits out of 5 lookups) and `llm_reduction_pct` should be ~80%.

== Exercise 4: Verify in the Inference Log

[source,bash,role="execute",subs=attributes+]
----
curl -s {healthcare_url}/api/v1/stats | jq '{total_requests, kv_cache_hit_rate}'
----

The `kv_cache_hit_rate` in the PostgreSQL inference log reflects all cached classifications across the system.

== What You Measured

Day 1: 100% LLM. Week 2: ~60% deterministic. Month 1: ~95% deterministic. At scale (1M+ records/month), this reduces LLM calls by 20x.

This is only possible at $0/token. Paying per token gives no incentive to cache — every record costs the same whether it's the first or the ten millionth. On Xeon 6, every cached classification frees inference capacity for records that actually need intelligence.
