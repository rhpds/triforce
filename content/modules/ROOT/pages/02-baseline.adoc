== Exercise 3: Benchmark Default Models

Run classification 3 times and take the median (shared infrastructure means variance):

[source,bash,role="execute",subs=attributes+]
----
for run in 1 2 3; do
  echo "--- Run $run ---"
  curl -s -X POST {healthcare_url}/api/v1/benchmark/run \
    -H "Content-Type: application/json" \
    -d '{"task": "classification", "text": "DISCHARGE SUMMARY: 72-year-old male with Type 2 Diabetes on Metformin 500mg.", "models": ["granite-2b-cpu", "qwen25-3b-cpu", "phi3-mini-cpu"]}' \
    | jq '[.results[] | {model, latency_ms}]'
done
----

Now the same models on NER (a harder task):

[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {healthcare_url}/api/v1/benchmark/run \
  -H "Content-Type: application/json" \

== Exercise 4: Find Your Slowest Step

Query the audit trail to find which pipeline step is your bottleneck. Fill in the task_type for your slowest step from Exercise 1:

[source,bash,subs=attributes+]
----
oc exec deploy/postgres -- psql -U triforce -d triforce -c \
  "SELECT model, task_type, COUNT(*) as calls,
          ROUND(AVG(latency_ms)) as avg_ms, MAX(latency_ms) as max_ms
   FROM inference_log
   WHERE task_type = 'YOUR_SLOWEST_STEP'
   GROUP BY model, task_type
   ORDER BY avg_ms DESC;"
----

Replace `YOUR_SLOWEST_STEP` with the task type that had the highest latency in your pipeline run (e.g. `summarize`, `classify`, `extract_entities`).
