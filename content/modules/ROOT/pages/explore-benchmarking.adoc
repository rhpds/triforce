= Benchmarking
:navtitle: Benchmarking

*Time: ~15 minutes* · *Terminal*

== The Question

How do different models compare on the same task? Where does CPU match GPU quality, and where does GPU pull ahead?

== How It Works

The benchmark suite runs the same prompt through multiple models and reports latency, output quality, and hardware tier side-by-side. 13 models are available — 9 on CPU (Xeon 6, $0/token) and 4 on GPU (Intel Gaudi, $/token).

== Exercise 1: List Available Models

[source,bash,role="execute",subs=attributes+]
----
curl -s {healthcare_url}/api/v1/benchmark/models | jq .
----

CPU models range from 350M to 8B parameters. GPU models range from 8B to 120B. Both tiers serve via the same MAAS/LiteLLM endpoint.

== Exercise 2: Compare Two CPU Models

Run classification on granite-2b vs qwen25-3b:

[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {healthcare_url}/api/v1/benchmark/run \
  -H "Content-Type: application/json" \
  -d '{"models": ["granite-2b-cpu", "qwen25-3b-cpu"], "task": "classification", "text": "DISCHARGE SUMMARY: 72-year-old male with Type 2 Diabetes on Metformin."}' \
  | jq '.results[] | {model, hardware, latency_ms, output}'
----

Compare latency and output. For classification, both models should agree on the document type — the question is which is faster.

== Exercise 3: Compare Across Tasks

Try the same models on NER:

[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {healthcare_url}/api/v1/benchmark/run \
  -H "Content-Type: application/json" \
  -d '{"models": ["granite-2b-cpu", "qwen25-3b-cpu"], "task": "ner", "text": "Patient on Aspirin 81mg, Clopidogrel 75mg, Metformin 1000mg. Diagnosed with Type 2 Diabetes and coronary artery disease."}' \
  | jq '.results[] | {model, latency_ms, output: .output[:200]}'
----

Different models excel at different tasks. Granite-2b is optimized for NER. Qwen25-3b handles longer summarization better.

== Exercise 4: Check the Fastest

[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {healthcare_url}/api/v1/benchmark/run \
  -H "Content-Type: application/json" \
  -d '{"models": ["granite-2b-cpu", "qwen25-3b-cpu", "phi3-mini-cpu"], "task": "classification", "text": "LAB REPORT: HbA1c 7.2%, fasting glucose 142 mg/dL."}' \
  | jq '{fastest: .fastest.model, fastest_ms: .fastest.latency_ms, slowest: .slowest.model, slowest_ms: .slowest.latency_ms}'
----

The benchmark response includes `fastest` and `slowest` summaries so you can quickly compare.

== The Intel Story

Benchmarking across the Intel compute spectrum — from 350M CPU models to 120B GPU models — reveals where CPU is sufficient and where GPU adds value. For 80% of enterprise tasks (classification, NER, fraud scoring), the CPU tier matches GPU quality at $0/token. Benchmarking proves it with measured numbers, not claims.

== What You Measured

* Side-by-side model comparison on the same task
* Latency differences across model sizes
* Where smaller CPU models match larger models on structured tasks
* The full model fleet available on Intel hardware (Xeon 6 + Gaudi)
