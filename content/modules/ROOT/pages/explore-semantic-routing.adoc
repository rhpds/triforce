= Semantic Router
:navtitle: Semantic Router

*Time: ~10 minutes* · *Terminal*

== The Question

How do you route each request to the right-sized model without making an LLM call to decide?

== How It Works

The vLLM Semantic Router uses sentence embeddings to classify request complexity in <2ms. No LLM call — just cosine similarity against pre-defined route anchors:

[cols="1,1,2",options="header"]
|===
| Route | Model | Tasks

| Simple | granite-2b-cpu (2B) | Classification, NER, labeling
| Medium | qwen25-3b-cpu (3B) | Summarization, extraction
| Complex | phi3-mini-cpu (3.8B) | Multi-step reasoning, differential diagnosis
|===

== Exercise 1: Classify a Simple Query

[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {semantic_router_url}/classify \
  -H "Content-Type: application/json" \
  -d '{"text": "Classify this clinical document as a discharge summary or lab report."}' | jq .
----

Look at `route`, `model`, and `confidence`. A simple classification task should route to the 2B model.

== Exercise 2: Classify a Complex Query

[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {semantic_router_url}/classify \
  -H "Content-Type: application/json" \
  -d '{"text": "Analyze the drug interactions between Warfarin, Aspirin, and Clopidogrel considering renal function and provide a differential diagnosis."}' | jq .
----

Compare the `route` and `confidence` to Exercise 1. Complex reasoning routes to the 3.8B model.

== Exercise 3: See All Routes

[source,bash,role="execute",subs=attributes+]
----
curl -s {semantic_router_url}/routes | jq '.[] | {name, model, params, anchor_count}'
----

Each route has anchor phrases that the embedding model compares against. More anchors = better classification accuracy.

== Exercise 4: Compare Router Latency to Inference

[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {semantic_router_url}/classify \
  -H "Content-Type: application/json" \
  -d '{"text": "Summarize this patient record."}' | jq '{route, model, router_ms: .latency_ms}'
----

The router decides in <2ms. The inference it routes to takes ~1-4s. Without the router, you'd either over-provision (use the 3.8B model for everything) or under-provision (use the 2B model and get poor quality on complex tasks).

== What You Measured

Embedding-based routing on CPU avoids an LLM call just to decide which LLM to call. The right model for the right task — decided in milliseconds, not seconds.
