== Exercise 2: Test the Router

The semantic router classifies request complexity using embeddings and decides which model and hardware should handle the request. Check the current routing configuration:

[source,bash,role="execute",subs=attributes+]
----
curl -s {semantic_router_url}/routes | jq .
----

Now send 5 texts of varying complexity and record where each one routes:

[source,bash,role="execute",subs=attributes+]
----
echo "--- Text 1: Simple ---"
curl -s -X POST {semantic_router_url}/classify \
  -H "Content-Type: application/json" \
  -d '{"text": "What type of document is this?"}' \
  | jq '{route, model, confidence}'
----

[source,bash,role="execute",subs=attributes+]
----
echo "--- Text 2: Short clinical ---"
curl -s -X POST {semantic_router_url}/classify \
  -H "Content-Type: application/json" \
  -d '{"text": "Patient diagnosed with hypertension. Started on Lisinopril 10mg daily."}' \
  | jq '{route, model, confidence}'

[source,bash,role="execute",subs=attributes+]

curl -s -X POST {semantic_router_url}/classify \
  -H "Content-Type: application/json" \
  -d '{"text": "72-year-old male with Type 2 Diabetes on Metformin 500mg and Lisinopril 10mg. Recent STEMI with PCI to RCA. Aspirin 81mg daily."}' \
  | jq '{route, model, confidence}'

== Exercise 3: Configure the Routing Threshold

The router's word count threshold determines when a request escalates from the simple route (small/CPU model) to the complex route (larger/GPU model). You are going to lower the threshold to make the router more aggressive about escalating.

First, back up the current configuration:

[source,bash,role="execute",subs=attributes+]
----
oc get configmap semantic-router-config -o jsonpath='{.data.routes\.yaml}' > /tmp/routes-backup.yaml
----

Check the current `word_count_thresholds.medium` value:

[source,bash,role="execute",subs=attributes+]
----
oc get configmap semantic-router-config -o jsonpath='{.data.routes\.yaml}' | grep -A2 word_count
----

Patch the threshold from 40 words to 25 words. This means more requests will be classified as complex:

[source,bash,subs=attributes+]
----
oc get configmap semantic-router-config -o json \
  | jq '.data["routes.yaml"] |= gsub("medium: 40"; "medium: 25")' \
  | oc apply -f -
----

Restart the semantic router to pick up the new configuration:

[source,bash,role="execute",subs=attributes+]
----
oc rollout restart deploy/semantic-router
oc rollout status deploy/semantic-router --timeout=120s
----

NOTE: The router takes ~2 minutes to restart because it loads an embedding model at startup. This is expected.

Now re-test the same 5 texts and see if any moved to a different route:

[source,bash,role="execute",subs=attributes+]
----
for text in \
  "What type of document is this?" \
  "Patient diagnosed with hypertension. Started on Lisinopril 10mg daily." \
  "72-year-old male with Type 2 Diabetes on Metformin 500mg and Lisinopril 10mg. Recent STEMI with PCI to RCA. Aspirin 81mg daily." \
  "Patient on Warfarin 5mg, Aspirin 81mg, Clopidogrel 75mg, and Metformin 500mg. Recent STEMI with PCI to RCA. CKD stage 3 with eGFR 42. Monitor INR closely and adjust anticoagulation therapy." \
  "Analyze the drug interactions between Warfarin and dual antiplatelet therapy considering the patients renal function with eGFR 42 and recent coronary stenting. Provide a risk-benefit assessment of continuing triple therapy versus de-escalating to dual therapy given the bleeding risk profile and stent thrombosis risk within the first 30 days post-PCI."; do
  echo "--- $(echo "$text" | head -c 50)... ---"
  curl -s -X POST {semantic_router_url}/classify \
    -H "Content-Type: application/json" \
    -d "{\"text\": \"$text\"}" \
    | jq '{route, model, confidence}'
done
----
