= Accessing Your Environment
:navtitle: Setup

== Learning Objectives

* Log in to the OpenShift cluster
* Verify all Triforce services are running
* Access the demo frontend

== Exercise 1: Log In to OpenShift

. Open a terminal and log in:
+
[source,bash,role="execute",subs=attributes+]
----
oc login --insecure-skip-tls-verify -u {user} -p {password} {openshift_api_url}
----

. Verify you're in the correct project:
+
[source,bash,role="execute"]
----
oc project triforce
----

== Exercise 2: Verify Services

. Check that all pods are running:
+
[source,bash,role="execute"]
----
oc get pods -n triforce
----
+
Expected — 10 pods in `Running` status:
+
[source]
----
NAME                               READY   STATUS
healthcare-agent-xxxxx             1/1     Running    (x2 replicas)
finserv-agent-xxxxx                1/1     Running    (x2 replicas)
orchestrator-xxxxx                 1/1     Running
mcp-gateway-xxxxx                  1/1     Running
semantic-router-xxxxx              1/1     Running
postgres-xxxxx                     1/1     Running
redpanda-xxxxx                     1/1     Running
frontend-xxxxx                     2/2     Running    (app + oauth proxy)
----

. Check service health:
+
[source,bash,role="execute",subs=attributes+]
----
curl -s {healthcare_url}/health | jq .
curl -s {finserv_url}/health | jq .
curl -s {orchestrator_url}/health | jq .
----

== Exercise 3: Access the Frontend

. Open the Triforce demo in your browser:
+
[source,subs=attributes+]
----
{frontend_url}
----

. You'll authenticate via Red Hat SSO, then see the demo start screen with Red Hat × Intel × IBM logos.

. Click to begin — the Triforce intro animates, then Act 00 reveals the story.

== Module Summary

* Logged in to OpenShift
* Verified all 10 Triforce pods are running
* Accessed the demo frontend via Red Hat SSO
