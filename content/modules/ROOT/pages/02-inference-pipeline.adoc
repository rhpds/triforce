= Healthcare Agent
:navtitle: Healthcare Agent

*Time: ~15 minutes* · *Terminal*

== The Question

Can a multi-step NLP pipeline run entirely on CPU with acceptable latency?

== How It Works

The healthcare agent runs a 4-node LangGraph pipeline. Each step uses a different model — the right size for the right task:

[cols="1,2,1",options="header"]
|===
| Step | What It Does | Model

| Classify
| Identify document type (discharge summary, lab report, etc.)
| qwen25-3b-cpu (3B)

| Extract Entities
| Find conditions, medications, procedures
| granite-2b-cpu (2B)

| Check Interactions
| Screen medications for drug-drug interactions
| MCP tool (FDA database — not an LLM call)

| Summarize
| Generate clinical summary with context from prior steps
| qwen25-3b-cpu (3B)
|===

The *Check Interactions* step is conditional — it only runs when 2+ medications are detected. Drug interaction data comes from a curated FDA / NIH RxNav database via MCP tool (~14ms lookup, not a ~3s LLM call).

== Exercise 1: Run the Pipeline

[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {healthcare_url}/api/v1/pipeline \
  -H "Content-Type: application/json" \
  -d '{"text": "DISCHARGE SUMMARY: 72-year-old male with Type 2 Diabetes on Metformin and Lisinopril. Recent STEMI with PCI to RCA. Started on Aspirin 81mg, Clopidogrel 75mg, Atorvastatin 80mg."}' | jq .
----

Look at the response:

* `classification` — document type
* `entities` — extracted medical entities with types
* `drug_interactions` — interactions found via MCP tool (this patient is on 5 medications)
* `summary` — generated clinical summary
* `inference_log` — per-step model and latency
* `total_ms` — end-to-end time

== Exercise 2: Inspect the Inference Log

Extract just the per-step timing:

[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {healthcare_url}/api/v1/pipeline \
  -H "Content-Type: application/json" \
  -d '{"text": "DISCHARGE SUMMARY: 72-year-old male with Type 2 Diabetes on Metformin and Lisinopril."}' \
  | jq '.inference_log[] | {node, model, latency_ms}'
----

Note which model ran each step and the latency. Every call is real inference on Intel Xeon 6.

== Exercise 3: Test the Conditional Skip

Run a patient with only one medication — the drug interaction step should be skipped:

[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {healthcare_url}/api/v1/pipeline \
  -H "Content-Type: application/json" \
  -d '{"text": "LAB REPORT: 45-year-old female on Metformin 500mg. HbA1c 7.2%, fasting glucose 142."}' \
  | jq '.inference_log[] | {node, model, latency_ms}'
----

Compare the step count to Exercise 2. The pipeline adapts to the input — no wasted inference.

== Exercise 4: Try Your Own Text

Write your own clinical note and see what the pipeline produces:

[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {healthcare_url}/api/v1/pipeline \
  -H "Content-Type: application/json" \
  -d '{"text": "YOUR CLINICAL TEXT HERE"}' | jq .
----

Try different document types: a progress note, a radiology report, a surgical summary. See how classification changes and which entities get extracted.

== Exercise 5: Check the Audit Trail

Every LLM call is logged to PostgreSQL with the model, task type, latency, and accelerator:

[source,bash,role="execute",subs=attributes+]
----
oc exec deploy/postgres -- psql -U triforce -d triforce -c \
  "SELECT model, task_type, latency_ms, accelerator FROM inference_log ORDER BY created_at DESC LIMIT 10;"
----

100% CPU, 0% GPU. Every inference call ran on Intel Xeon 6.

== What You Measured

* End-to-end pipeline latency on CPU
* Per-step model selection and timing
* Conditional pipeline behavior (skipping unnecessary steps)
* Full audit trail in PostgreSQL
