= Path D: Scale & Cost
:navtitle: Path D: Scale & Cost

== Objective

Test how the system behaves under increasing concurrent load, find the saturation point, and build a cost model from your measured data.

== Exercise 1: Scale Up and Down

Check the current replica count, then test at 1, 2, and 3 replicas:

[source,bash,role="execute",subs=attributes+]
----
echo "Current replicas:"
oc get deploy healthcare-agent -o jsonpath='{.spec.replicas}'
echo
----

Scale to 1 replica (intentionally under-provision):

[source,bash,role="execute",subs=attributes+]
----
oc scale deploy healthcare-agent --replicas=1
oc rollout status deploy healthcare-agent --timeout=60s
----

Run a single baseline request at 1 replica:

[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {healthcare_url}/api/v1/pipeline \
  -H "Content-Type: application/json" \
  -d '{"text": "Patient with hypertension on Lisinopril 10mg.", "skip_cache": true}' \
  | jq '\{total_ms}'
----

Now scale to 3 replicas:

[source,bash,role="execute",subs=attributes+]
----
oc scale deploy healthcare-agent --replicas=3
oc rollout status deploy healthcare-agent --timeout=60s
----

Run the same request:

[source,bash,role="execute",subs=attributes+]
----
curl -s -X POST {healthcare_url}/api/v1/pipeline \
  -H "Content-Type: application/json" \
  -d '{"text": "Patient with hypertension on Lisinopril 10mg.", "skip_cache": true}' \
  | jq '\{total_ms}'
----

== Exercise 2: Concurrency Sweep

Keep 3 replicas running. Fire 1, 5, 10, and 20 concurrent requests and measure wall-clock time and per-request average:

[source,bash,role="execute",subs=attributes+]
----
for concurrent in 1 5 10 20; do
  echo "=== $concurrent concurrent ==="
  start=$(date +%s%N)
  for i in $(seq 1 $concurrent); do
    curl -s -X POST {healthcare_url}/api/v1/pipeline \
      -H "Content-Type: application/json" \
      -d '{"text": "Patient with hypertension on Lisinopril 10mg.", "skip_cache": true}' \
      -o /dev/null &
  done
  wait
  end=$(date +%s%N)
  total_ms=$(( (end - start) / 1000000 ))
  avg_ms=$(( total_ms / concurrent ))
  echo "  Wall clock: $\{total_ms}ms  |  Avg per request: $\{avg_ms}ms"
  echo
done
----

Check pod resource usage during or after the sweep:

[source,bash,role="execute",subs=attributes+]
----
oc adm top pods -l app=healthcare-agent 2>/dev/null || echo "Metrics server not available -- check pod status with: oc get pods -l app=healthcare-agent -o wide"
----

== Exercise 3: Find the Saturation Point

Your saturation point is the concurrency level where per-request latency exceeds *2x your single-request baseline*.

[sidebar]
****
*Calculate:*

* Single-request baseline (1 concurrent): ___ ms
* 2x threshold: ___ ms
* Saturation point (concurrency where avg exceeds 2x): ___ concurrent
* At saturation, throughput = concurrent / (wall_clock_ms / 1000) = ___ requests/second
* Requests per hour at saturation: ___ (× 3600)
****

Now scale back to 1 replica and re-run the sweep at 1 and 5 concurrent:

[source,bash,role="execute",subs=attributes+]
----
oc scale deploy healthcare-agent --replicas=1
oc rollout status deploy healthcare-agent --timeout=60s
for concurrent in 1 5; do
  echo "=== $concurrent concurrent (1 replica) ==="
  start=$(date +%s%N)
  for i in $(seq 1 $concurrent); do
    curl -s -X POST {healthcare_url}/api/v1/pipeline \
      -H "Content-Type: application/json" \
      -d '{"text": "Patient with hypertension on Lisinopril 10mg.", "skip_cache": true}' \
      -o /dev/null &
  done
  wait
  end=$(date +%s%N)
  total_ms=$(( (end - start) / 1000000 ))
  avg_ms=$(( total_ms / concurrent ))
  echo "  Wall clock: $\{total_ms}ms  |  Avg per request: $\{avg_ms}ms"
done
----

== Learning Outcomes

By completing this module, you should now understand:

* More replicas help concurrent throughput but not single-request latency
* The saturation point depends on both agent pod capacity and MAAS model serving
* How to calculate records/hour, replicas needed, and monthly cost from measured data
* The cost advantage of CPU inference at scale -- $0/token vs cloud API pricing
