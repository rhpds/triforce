= Module 6: Scale and Cost
:navtitle: Scale and Cost

== Objective

Test how the system behaves under concurrent load and build a cost model for your workload at scale.

== Exercise 1: Scale Replicas

Check the current replica count:

[source,bash,role="execute",subs=attributes+]
----
oc get deploy healthcare-agent -o jsonpath='{.spec.replicas}'
echo " replicas"
----

Scale to 3 replicas:

[source,bash,role="execute",subs=attributes+]
----
oc scale deploy healthcare-agent --replicas=3
oc rollout status deploy healthcare-agent --timeout=60s
----

== Exercise 2: Concurrent Load

Fire 5 concurrent pipeline requests and compare latency to your baseline:

[source,bash,role="execute",subs=attributes+]
----
echo "Starting 5 concurrent requests..."
for i in 1 2 3 4 5; do
  (time curl -s -X POST {healthcare_url}/api/v1/pipeline \
    -H "Content-Type: application/json" \
    -d '{"text": "Patient with hypertension on Lisinopril 10mg.", "skip_cache": true}' \
    | jq -r '"Request '$i': \(.total_ms)ms"') &
done
wait
echo "All done."
----

Check pod resource usage:

[source,bash,role="execute",subs=attributes+]
----
oc adm top pods -l app=healthcare-agent 2>/dev/null || echo "Metrics server not available -- check pod status with: oc get pods -l app=healthcare-agent"
----

Restore to 2 replicas:

[source,bash,role="execute",subs=attributes+]
----
oc scale deploy healthcare-agent --replicas=2
----

* Single request latency (from baseline): ___ ms
* Average latency under 5 concurrent: ___ ms
* Degradation: ___x slower
* Where is the bottleneck? (agent pods or model serving?) ___
****

== What You've Built

You now have a decision framework with your own measured data:

. *Model assignment* -- which model for which task (Module 3)
. *Optimization stack* -- which techniques apply to your workload (Module 4)
. *Routing policy* -- which tasks stay on CPU, which go to GPU (Module 5)
. *Cost model* -- what this costs at scale vs alternatives (this module)
