= Go Deeper: Load Testing
:navtitle: Load Testing

== What You'll Do

Generate concurrent load at increasing levels and observe how latency degrades. Find the saturation point.

== Exercise 1: Single Request Baseline

Time a single request:

[source,bash,role="execute",subs=attributes+]
----
time curl -s -X POST {healthcare_url}/api/v1/pipeline \
  -H "Content-Type: application/json" \
  -d '{"text": "Patient with hypertension on Lisinopril 10mg.", "skip_cache": true}' \
  | jq -r '"Latency: \(.total_ms)ms"'
----

== Exercise 2: Concurrency Sweep

Run 1, 5, 10, and 20 concurrent requests and record the average latency:

[source,bash,role="execute",subs=attributes+]
----
for concurrent in 1 5 10 20; do
  echo "=== $concurrent concurrent ==="
  start=$(date +%s%N)
  for i in $(seq 1 $concurrent); do
    curl -s -X POST {healthcare_url}/api/v1/pipeline \
      -H "Content-Type: application/json" \
      -d '{"text": "Patient with hypertension on Lisinopril.", "skip_cache": true}' \
      -o /dev/null &
  done
  wait
  end=$(date +%s%N)
  total_ms=$(( (end - start) / 1000000 ))
  avg_ms=$(( total_ms / concurrent ))
  echo "  Wall clock: ${total_ms}ms  |  Avg per request: ${avg_ms}ms"
  echo
done
----
