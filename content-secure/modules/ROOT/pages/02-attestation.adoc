== Learning Objectives

* Understand how Trustee remote attestation works
* See how secrets are gated by hardware verification

== How Attestation Works

```
Pod starts with runtimeClassName: kata-cc
    │
    ├── Kata creates VM with TDX memory encryption
    │
    ├── Agent calls Trustee KBS: "Give me the API key"
    │
    ├── Trustee asks: "Prove you're running in TDX"
    │   └── Hardware attestation via TDX REPORT
    │
    ├── TDX verified → Secret released → Inference works
    │
    └── TDX NOT verified → Secret denied → Inference fails
```

The MAAS API key that controls all inference is *hardware-gated*. No TDX, no key, no inference.
