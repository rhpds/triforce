= Verify TDX & Operators
:navtitle: Setup

== Exercise 1: Check Node TDX Capability

[source,bash,role="execute"]
----
oc get nodes -o json | jq '.items[] | {name: .metadata.name, tdx: .status.capacity["intel.com/tdx"]}'
----

Nodes with `tdx: "1"` support Trust Domain Extensions.

== Exercise 2: Verify Operators

[source,bash,role="execute"]
----
oc get csv -n openshift-sandboxed-containers-operator | grep -i sandboxed
oc get csv -n trustee-operator-system | grep -i trustee
----

* **Sandboxed Containers** — provides the `kata-cc` RuntimeClass
* **Trustee** — provides KBS (Key Broker Service) for attestation-gated secrets

== Exercise 3: Verify kata-cc RuntimeClass

[source,bash,role="execute"]
----
oc get runtimeclass kata-cc -o yaml | head -10
----

This is the single line that changes everything: `runtimeClassName: kata-cc`.

== Module Summary

* Verified TDX-capable nodes
* Confirmed Sandboxed Containers + Trustee operators installed
* Found the `kata-cc` RuntimeClass
