= Triforce Secure — Confidential AI on Intel Xeon 6
:navtitle: Welcome

== Why Confidential AI?

Healthcare records contain PHI. Financial transactions contain PII. When AI inference processes this data, it exists unencrypted in memory — visible to anyone with cluster admin access.

Intel TDX (Trust Domain Extensions) solves this: hardware-encrypted memory isolation. The CPU encrypts all data in the Trust Domain. Not even the cluster admin or hypervisor can read it.

In this lab you will:

* Verify Intel TDX is active on Xeon 6 nodes
* Configure Trustee for remote attestation
* Deploy AI agents inside Confidential Containers (kata-cc runtime)
* Prove memory encryption is active (`dmesg | grep tdx`)
* Run inference — same granite-2b-cpu model, same quality, hardware-encrypted
* See that the API key is only released to TDX-attested pods
