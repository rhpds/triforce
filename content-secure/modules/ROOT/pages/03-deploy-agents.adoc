= Deploy with One Line
:navtitle: Deploy

== The Entire Difference

Standard deployment vs confidential deployment — one line:

[source,yaml]
----
spec:
  template:
    spec:
+     runtimeClassName: kata-cc    # ← this is the only change
      containers:
        - name: agent
          image: triforce/healthcare-agent:latest  # same image
----

Same container image. Same application code. Same AI model. Same API endpoints. Different runtime — hardware-encrypted memory.

== Exercise 1: Verify the Runtime Class

[source,bash,role="execute"]
----
oc get deploy healthcare-agent -n triforce -o jsonpath='{.spec.template.spec.runtimeClassName}'
----

If confidential is enabled, this returns `kata-cc`. If standard, it's empty.

== Exercise 2: Compare the Deployment

[source,bash,role="execute"]
----
oc get deploy healthcare-agent -n triforce -o yaml | grep -A 2 "runtimeClassName\|image:"
----

The image is identical. The code is identical. The model is identical. Only the runtime changed.

== Exercise 3: The Helm Toggle

[source,bash,role="execute"]
----
helm get values triforce -n triforce | grep -A 2 confidential
----

`confidential.enabled: true` → Helm injects `runtimeClassName: kata-cc` into all agent deployments.

== Module Summary

* Saw the one-line change: `runtimeClassName: kata-cc`
* Verified the container image, code, and model are unchanged
* Understood the Helm toggle
