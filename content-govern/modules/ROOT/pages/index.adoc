= Triforce Govern — AI Agent Governance with Kagenti
:navtitle: Welcome

== The Problem

When you have multiple AI agents — each calling LLMs, invoking tools, processing sensitive data — who controls them? Who decides which agent can call which tool? How do you audit every decision?

IBM Kagenti answers this: Kubernetes-native agent governance with A2A discovery, MCP tool federation, SPIFFE zero-trust identity, and full audit trails.

In this lab you will:

* Enroll AI agents as Kagenti AgentRuntimes (K8s CRDs)
* See agents auto-discovered via A2A protocol
* Verify SPIFFE workload identity between agents
* Configure MCP tool governance policies
* Query the audit trail for every inference and tool call
