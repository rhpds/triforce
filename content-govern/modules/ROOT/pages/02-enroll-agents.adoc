= Agents as K8s Resources
:navtitle: Agent Discovery

== Exercise 1: Discover Agents via Orchestrator

[source,bash,role="execute",subs=attributes+]
----
curl -s {orchestrator_url}/api/v1/agents | jq '.agents[] | {name, url, status, skills: [.skills[].name]}'
----

The orchestrator discovers agents automatically via A2A protocol. No manual wiring.

== Exercise 2: Read an Agent Card

[source,bash,role="execute",subs=attributes+]
----
curl -s {healthcare_url}/.well-known/agent-card.json | jq '{name, description, version, protocolVersion, provider, skills: [.skills[] | {id, name}]}'
----

Every agent publishes its capabilities as a standard JSON document. Skills, version, protocol — all discoverable.

== Exercise 3: Both Agents

[source,bash,role="execute",subs=attributes+]
----
echo "=== Healthcare ===" && curl -s {healthcare_url}/.well-known/agent-card.json | jq '{name, skills: [.skills[].id]}'
echo "=== FinServ ===" && curl -s {finserv_url}/.well-known/agent-card.json | jq '{name, skills: [.skills[].id]}'
----

2 agents, 6 skills, zero manual configuration. When a new agent deploys with the right labels, it's discoverable within seconds.
